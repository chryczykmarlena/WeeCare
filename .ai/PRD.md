# Product Requirements Document (PRD): WeeCare - Child Medical History Tracker

## 1. Project Overview
WeeCare is a child medical history tracker that allows parents to keep track of their children's medical history in one place. It provides a secure and easy way to store and access important medical information, such as medication schedules and visit history. Additional features include contacts and opinions about doctors.

## 2. Goals & Objectives

- **Goal 1:** Centralize all doctor appointments and prescriptions records for all children in one place.
- **Goal 2:** My own opinions about doctors.
- **Goal 3:** Easy access to medical history.

## 3. User Personas
There is only one user for this app: the parent/guardian.

### 3.1 Primary User: Parent / Guardian
- **Needs:** Quick entry of data, easy retrieval, web access, see doctors' appointments history and prescriptions history. Additionally I should be able to write my own opinion about the doctor and see it in the app.
- **Pain Points:** Scattered paper records, forgetting medication times and doctors' appointments, managing multiple kids. 

## 4. Core Features (Functional Requirements)

### 4.1 Child Profiles
- [ ] Create/Edit profiles for each child (Name, DOB, Allergies).
- [ ] Upload a profile picture.

### 4.2 Medical Records
- [ ] **Medication Log:** Current medications and history.
- [ ] **Visit History:** Log doctor visits, diagnosis, and notes.

### 4.3 Document Storage
- [ ] Upload photos/PDFs of lab results or prescriptions.

### 4.4 Notes about doctors
- [ ] Write my own notes about doctors.

### 4.5 Contacts
- [ ] Add contacts for doctors.

## 5. User Interface (UI) & User Experience (UX)

- **Theme:** Minimalist
- **Platform:** Web-first

## 6. Technical Requirements & Security
The app will be created and used by me.

- **Data Privacy:** I will not share any data with third parties.
- **Authentication:** Login via email/password and registration via email.
- **Tech Stack:** Next.js and a secure database.
